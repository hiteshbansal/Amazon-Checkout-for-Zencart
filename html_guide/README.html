Introduction

The Checkout By Amazon plugin v1.0 for Zen Cart 1.3.9e supports the following features

  1. Checkout By Amazon as a checkout option in the "shopping cart" page, Amazon 1-Click and Express checkout, Payphrase checkout Integration.
  2. Order Management (Ship Order, Cancel Order, Refund Order)through your website's admin portal. You can also see the order status history of a particular order on Amazon's end in your Zen Cart admin. NOTE: Orders should be managed only through your website. DO NOT use seller central for the same.
  3. Instant Order Notifications through IOPN.
    • Inventory update as soon as an order is placed.
    • Update as soon as a Buyer cancels an order.
    • Update as soon as a Amazon cancels an order due to Late shipment or risk related reasons.
  4. Send dynamic shipping rates at an order level using callbacks from different shipping carriers like Fedex, UPS, USPS, Flat rate etc, configured on your Zen Cart admin UI.
    NOTE: If you send more than 24 shipping methods, the buyer can choose from a maximum of 24 shipping methods (filtered) in “Place your Order” page of the ”Checkout With Amazon” pipeline.
  5. Dynamic tax rates can be sent at an order level using callbacks based on the delivery address of the buyer.
  6. Ability to add Custom data required by merchants via custom data modules.

    IMPORTANT: You will require a *https* endpoint to enable functionalities 3, 4 and 5.

You can also set promotion codes for orders that will be processed through Checkout By Amazon. Configuration help.

Configuration Options

Assuming that you have installed, you can configure the Checkout By Amazon button based on your requirements. You have the following options which inturn indicate different levels of integration of Checkout By Amazon with your website. Please go through the options below and choose the one that is best for you.

  1. Option 1: CBA Button only on your Website - Simple Integration - If you want to just enable Checkout By Amazon as a checkout option (in the "shopping cart" page), please follow the steps given in this option.
    IMPORTANT: If you go with this option, you will have to manage orders (fulfillment, cancel, refund etc), shipping rates and tax rates from Seller Central. [These actions CANNOT be done from Zen Cart admin]

  2. Option 2: CBA Button + Order Management from your Website's Admin - Standard Integration - If you want to enable Checkout By Amazon as a checkout option (in the "shopping cart" page) and also manage orders (Ship Order, Cancel Order, Refund Order) through your website's admin portal., please follow the steps given in this option.
    NOTE:     If you go with this option and wish to enable instant order status notifications using IOPN, you will require a *https* enabled endpoint.
    IMPORTANT: If you go with this option, you will have to manage shipping rates and tax rates from Seller Central. [These actions CANNOT be done from Zen Cart admin]

  3. Option 3: CBA Button + Order Management from your Website's Admin + Dynamic Shipping and Tax calculation - Advanced Integration - If you want to enable the complete suite of functionality offered by this plug-in like, Checkout By Amazon as a checkout option (in the "shopping cart" page), manage orders (Ship Order, Cancel Order, Refund Order) through your website's admin portal and also send dynamic shipping and tax rates at the time of order placement., please follow the steps given in this option.
    NOTE:     If you go with this option, you will require a *https* enabled endpoint for callback (dynamic shipping and tax rates) and IOPN (instant order status notifications).

For all of the above options, you can also set promotion codes for orders that will be processed through Checkout By Amazon. Configuration help.

Release Notes

  1. It is strongly encouraged to have your AWS key ready to enable signed orders. Unsigned orders are very vulnerable to fraud. You can setup your AWS key in http://sellercentral.amazon.com > Integration > AWS Access Key.
  2. Note that Checkout by Amazon standards do not allow weights of products to be 0 at this time. Thus, a weight tag = 0 will be removed from the XML cart before the order is processed.
  3. The orders that are processed through Checkout by Amazon will appear in Zen Cart if you enable order management. Currently orders that are not in shippable state are not displayed in the Zen Cart order management UI if IOPN endpoint is not set. For information on how to enable post order feeds, click here.
  4. Checkout by Amazon plug-in allows you to calculate shipping rates (using shipping modules in Zen Cart) and tax rates dynamically when the order is being placed by the customer. You need to ensure that your desired shipping modules and right tax settings are configured in Zen Cart. The configured shipping modules and tax settings will be used to calculate tax and shipping rates via Callback.
  6. The price that is included in the XML carts sent to Checkout by Amazon from Zen Cart includes promotion prices set in Zen Cart, .i.e. the promotion has already been applied to the item. If you, the seller, also configure promotions in Seller Central, these will be applied in addition to the promotion from Zen Cart.
  7. Please note that the SKU, Title, & Description fields of products may be truncated when building the Checkout by Amazon cart. This is to comply with the standards set by Checkout by Amazon. Please
    see the Checkout by Amazon documentation in "LINKS" section below.
  8. If you wish to create your own success return page and use the popup order summary from Checkout by Amazon, you will need to include the necessary scripts.
